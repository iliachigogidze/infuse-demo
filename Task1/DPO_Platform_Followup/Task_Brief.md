# DPO Platform — Follow-Up Task Brief

## Context
Follow-up submission for the DPO Platform assignment. The goal is to demonstrate the approach to several operational scenarios from a **process, data, and system** perspective.

---

## Scenarios to Address

### 1. Lead per Company Limit
How to enforce a maximum number of leads per company within a campaign.

### 2. Duplicate Leads from Different Vendors
How to prevent or detect duplicate leads submitted by multiple vendors for the same campaign.

### 3. Client Requirement Change (24h Delivery Limitation)
If a client changes requirements, but previously approved leads must still be delivered within 24 hours using the old criteria:
- How to manage leads already in the pipeline.
- How to separate old vs. new criteria across all teams and platforms.

### 4. Cross-Channel Contacts
How to ensure the same lead is not contacted for the same campaign across different channels (e.g., DPO and email marketing).

### 5. ABM List Change History
Clients may update their target account list, and new versions may include accounts from previous files. How to:
- Track changes and versions.
- Manage overlaps.
- Avoid reprocessing the same accounts.

---

## Deliverables
1. A clear explanation of the proposed approach for each scenario.
2. Updated **TO BE** workflow addressing end-to-end flow, all platforms involved, and roles and responsibilities in the process.
3. Clear visuals to support the solution (e.g., interface sketch, dashboard concept) to help stakeholders understand how the process would work in practice.
4. Any rules, validations, or system logic required.
5. 2–3 proposed improvements that enhance efficiency, accuracy, or user experience (out-of-the-box ideas).
6. Up to **one page** short description of how the task was handled, which tools were used, and any AI or other tools used.
